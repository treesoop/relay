## Key finding — `echo "$JSON" \| jq` is unsafe

When a curl response contains JSON-escaped `\n` inside a string value,
piping it through `echo "$VAR"` converts the escape to a literal newline,
breaking re-parsing. Fix: always use `printf '%s' "$VAR"`. Applied to
`adapters/claude/commands/upload.md` (other commands already used
`printf` or direct curl-to-jq pipes).
